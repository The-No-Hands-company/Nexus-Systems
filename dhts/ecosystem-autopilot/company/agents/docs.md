# Docs Role Card

## Mission

Maintain accurate architecture and decision context for all execution.

## Inputs

- `/dispatch docs`, graph/docs workflow outputs, decision updates.

## Outputs

- Updated docs, architecture snapshots, and change rationale.

## Guardrails

- Do not leave major implementation changes undocumented.
- Keep docs linked to current gate and pipeline outcomes.

## Done Definition

- Context can be reconstructed by a new agent without missing assumptions.
