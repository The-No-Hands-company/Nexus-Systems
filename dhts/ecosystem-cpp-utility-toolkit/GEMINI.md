# GEMINI.md: Your Guide to the Nexus Systems C++ Utility Toolkit

This file provides a comprehensive overview of the Nexus Systems C++ Utility Toolkit, its architecture, and development conventions.

## Project Overview

The Nexus Systems C++ Utility Toolkit is a professional-grade, enterprise-level diagnostic and development toolkit for modern C++23 projects. It provides a comprehensive set of utilities for various aspects of C++ development, including memory management, crash handling, logging, profiling, and more.

### Key Features

*   **Comprehensive:** Over 500 utilities across 67 categories.
*   **Modern:** Built for C++23.
*   **Cross-Platform:** Supports Linux, Windows, and macOS.
*   **Modular:** Use only the utilities you need.
*   **Zero-Cost Abstractions:** Diagnostics can be compiled out for release builds.

### Architecture

The toolkit is designed with a modular architecture. Each utility is implemented as a singleton and can be accessed through a static `instance()` method. The utilities are organized into categories, which are represented as subdirectories in the `include/nexus/utility` and `src/utility` directories.

A significant portion of the source code is generated by Python scripts located in the `scripts` directory. These scripts define the structure and boilerplate code for the utilities, ensuring consistency and reducing manual effort.

## Building and Running

The project uses CMake as its build system.

### Building the Library

1.  **Configure CMake:**
    ```bash
    cmake -B build -S .
    ```

2.  **Build the library:**
    ```bash
    cmake --build build
    ```

This will build the `nexus_utility_toolkit` static library in the `build/src/utility` directory.

### Running the Example

The project includes an example usage file at `src/utility/example_usage.cpp`. To build and run the example, you will need to create a `CMakeLists.txt` file that links against the `nexus_utility_toolkit` library.

## Testing

The project includes a suite of verification tests in the `tests` directory. Each `phaseX_verification.cpp` file corresponds to a set of utilities and acts as a self-contained test runner.

To run a test, compile and execute the corresponding verification file. For example, to run the phase 1 tests:

```bash
g++ -std=c++23 -Iinclude tests/phase1_verification.cpp -o phase1_test
./phase1_test
```

## Development Conventions

### Coding Style

*   **Namespaces:** All code is within the `nexus::utility` namespace, with sub-namespaces for each category (e.g., `nexus::utility::memory`).
*   **Classes:** Class names are in `PascalCase`.
*   **Functions and Variables:** Function and variable names are in `camelCase` or `snake_case`.
*   **Macros:** Macros are in `UPPER_SNAKE_CASE` and prefixed with `NEXUS_`.
*   **Includes:** Include guards are `#pragma once`.

### Source Code Generation

The source code for the utilities is generated by Python scripts in the `scripts` directory.

*   `scripts/generate_utilities.py`: Defines the basic set of utilities and generates their source files.
*   `scripts/complete_utilities_manifest.py`: Contains a comprehensive list of all utilities.
*   `scripts/generate_all_utilities.py`: A master script that combines the other two scripts to generate all utility files.

When adding a new utility, you should add it to one of these Python scripts and then run `scripts/generate_all_utilities.py` to generate the source files.
