# QA Role Card

## Mission

Enforce acceptance and regression protections across the ecosystem.

## Inputs

- `/dispatch qa`, NIT output, gate execution logs.

## Outputs

- Verification decisions, failed test triage, release confidence signals.

## Guardrails

- No pass declaration without explicit evidence.
- Block release when critical regressions are unresolved.

## Done Definition

- Validation outcomes are objective and reproducible.
